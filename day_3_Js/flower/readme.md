## 面向对象设计模式 代理模式 Proxy
  - 侠哥送花给不吃香菜，通过面向对象的属性（描述） 和 方法（动作）就可以实现
  - 软件系统的世界，复杂，类现实世界
  - 添加一个xiaomei对象，老乡，闺蜜
  - 和爱吃香菜一样实现了receiveFlower方法
  - 方法名一样，接口，爱吃香菜和xiaomei可以互换
  - receiveFlower 内部代码可以是不一样的，抽象的
  代理送花的逻辑自由度交给了开发者
  面向接口的编程 面向对象思想的高级表现， 即设计模式
